Features that make omp4j so cool

Why to choose omp4j instead of some similar project? What does omp4j excel in? Keep reading and learn everything important about the advantages of our preprocessor.

Portability run everywhere

omp4j runs on all commonly used JVMs. Try for example OracleJDK7/8 or OpenJDK6/7. We developed the preprocessor in order to run everywhere.

No runtime dependencies really!

Compile your code - pack a JAR - deploy. The preprocessed code does not require any installed library. It will run on servers, desktops, mobile phones and tables. Maybe even on your washing machine!

Java 8 support

Use lambdas and other cool Java 8 constructs as you will. No limitations. However, keep in mind that we use ANTLR JAVA 8 grammar for code parsing and it might be buggy.

High scalability as much as possible

Do you have independent tasks? Enjoy almost linear speedup up to 24 cores. Do you have even more cores machine? Great! Get significantly better speedup than related projects (e.g. JOMP) or almost the same as the original C++ OpenMP. Your only enemy remains Amdahl's law.

IDE support literally every IDE

Replace regular javac with omp4j in your IDE. Since omp4j accepts the same options as javac, everything will work. Your IDE even doesn't need to know that you use omp4j.

Library-like behavior run it from your application

Employ omp4j as a library from your Java/Scala project. It is easy! This webpage runs entirely on Scala and the demo is supported via the omp4j API.
