Features and other cool stuff

Why to choose omp4j instead of some similar project? What does omp4j excels in?

Portability
omp4j runs on all commonly used JVMs. Try for example OracleJDK7/8 or OpenJDK6/7.
No runtime dependencies
Compile your code. Pack a JAR. Deploy everywhere. The preprocessed code does not require any installed library.
Java 8 support
Use lambdas and other cool Java 8 constructs as you will. No limitations.
High scalability
Independent tasks? Enjoy almost linear speedup up to 24 cores. Even more cores? Get significantly better speedup than related projects or almost the same as the original OpenMP.
IDE support
You replace regular javac with omp4j in your IDE. omp4j accepts the same options as javac, so the IDE even doesn't need to know!
Library support
Employ omp4j as a library from your Java/Scala project. This webpage runs entirely on Scala and support cod translation via the API.

Quick start in 10 seconds

Interested in how simply omp4j may be used? Read through the 10 seconds examples.

Type a comment in front of a code block and let the preprocessor do the parallel invocation for you. Simple as that. 

// omp parallel threadNum(4)
{
	System.out.print("hello ")
}

/* output */
hello hello hello hello

Alternatively let the for-loop run in parallel. Can you think of a simpler solution? 

// omp parallel for
for (int i = 0; i < 10; i++) {
	System.out.print(i)
}

/* output */
2 0 6 7 1 9 5 3 4 8
